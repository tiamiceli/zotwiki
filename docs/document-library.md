# ZotWiki — Document Library

Organized by location in the repository. Each entry lists the file, the role
that primarily owns or executes it, and its purpose.

**Roles:** `human` = project owner / operator; `planner` = architect/decision
maker; `coder` = implementation author; `tester` = test suite author.

## `src/zotwiki/` — production source (coder-owned)

| File | Owner | Purpose |
|---|---|---|
| `__init__.py` | coder | Package init; exports `__version__ = "0.1.0"`. |
| `__main__.py` | coder | Enables `python -m zotwiki`; calls `sys.exit(main())`. |
| `errors.py` | coder | Full `ZotWikiError` exception hierarchy: `ZoteroError` (and subclasses `ZoteroUnavailableError`, `ItemNotFoundError`, `CitekeyNotFoundError`, `FulltextNotFoundError`, `CollectionNotFoundError`), `ArticleSchemaError`, `PageParseError`, `VaultError`. |
| `models.py` | coder | Frozen dataclasses: `SourceItem`, `Section`, `Quote`, `Claim`, `Article`, `Contradiction`. Also `normalize_text()` (NFKC + quote/dash normalization + casefold + whitespace collapse) used by the auditor and merge logic. |
| `zotero.py` | coder | `ZoteroStore` protocol and `HTTPZoteroStore` implementation. Handles all Zotero local API HTTP calls: search, get, fulltext (two-step probe: parent then child attachments per REQ-045), resolve, add, collection\_items, children. Includes retry/backoff logic and citekey generation for `add`. |
| `llm.py` | coder | `LLMClient` protocol (`complete(prompt) -> str`). `ClaudeCodeLLMClient` shells out to `claude --print`. `parse_article_json()` validates and parses the LLM's JSON output against the SS5.2 schema. `article_to_json_dict()` serializes an `Article` for round-trips and update-mode prompts. |
| `compiler.py` | coder | `Compiler` class: fetches items from the store, builds an LLM prompt (via `_build_prompt`), calls the LLM, and parses the result into a `CompileResult`. `merge_articles()` implements the SS7.2 never-clobber merge for update mode. |
| `publisher.py` | coder | `VaultPublisher`: writes compiled articles to `.md` files (`publish`), maintains `Index.md` (`_regenerate_index`), and appends to `Contradictions.md` (`publish_contradictions`). Also `render_page()` (canonical byte-exact page format) and `parse_page()` (round-trip parser). |
| `auditor.py` | coder | `Auditor`: reads all vault pages and reports violations (`AuditReport`, `Violation`). Checks: unparseable pages, unresolved citekeys, quotes not found in fulltext, broken wiki-links, orphan pages, stale index entries, missing references. |
| `ask.py` | coder | `ask()`: reads all vault entity pages, sends them with a question to the LLM, validates the answer JSON (cited pages and citekeys must exist in the vault), and returns an `Answer` with source references. |
| `syncer.py` | coder | `Syncer`: iterates a Zotero collection, skips items without citekeys and pages that already exist (unless `--update`), compiles new items, publishes pages, and returns a `SyncReport`. |
| `cli.py` | coder | `main(argv, *, store, llm)`: the CLI entry point. Parses arguments (argparse), dispatches to `ingest` / `compile` / `audit` / `ask` / `sync` handlers, maps exceptions to exit codes (0/1/2), and writes structured stdout/stderr per `contract.md §9`. The `store=` and `llm=` keyword arguments are the permanent injection seam for tests. |

## `tests/` — test suite (tester-owned)

Testers read only `docs/contract.md` and `docs/requirements.md`. They never
read source files. The coder never writes here.

|---|---|---|
